# Implemented Pages & Functions (this MVP)

## Public
- `/` → redirects to default locale
- `/[locale]` — Home (hero, live stats, services, why Alexandria, news, partners)
- `/[locale]/services` — full service catalog (DB-driven)
- `/[locale]/students` — student portal (universities, scholarships, internships, jobs, events)
- `/[locale]/citizens` — citizen services overview
- `/[locale]/business` — business & trade opportunities (sector/country filter)
- `/[locale]/investment` — investment opportunities
- `/[locale]/directory` — Alexandria smart directory (category filter)
- `/[locale]/news` — news & events
- `/[locale]/faq` — FAQ with search
- `/[locale]/emergency` — SOS / emergency case submission
- `/[locale]/translate` — translation center
- `/[locale]/search` — global debounced search across services/news/directory/universities/companies/faq/events
- `/[locale]/proposal` — Strategic Proposal (ambassador-facing)
- `/[locale]/terms`, `/privacy`, `/cookies`, `/accessibility` — legal pages (AR/EN/VI)
- `/[locale]/auth/login`, `/[locale]/auth/register`

## Authenticated
- `/[locale]/dashboard` — personal dashboard (digital ID/QR, requests, appointments, documents,
  notifications, quick links)
- `/[locale]/requests/new` — submit a new service/case request
- `/[locale]/appointments` — browse and book appointment slots
- `/[locale]/documents` — upload, view, and delete personal documents
- `/[locale]/messages` and `/[locale]/messages/[threadId]` — support messaging threads
- `/[locale]/notifications` — notification center with mark-as-read / mark-all-as-read
- `/[locale]/complaints` — submit a complaint or suggestion (optionally anonymous)
- `/[locale]/settings` — change password, deactivate ("delete") account

## Admin (server-guarded by role/permission)
- `/[locale]/admin` — KPI overview + recent requests table + links to sub-modules & CSV export
- `/[locale]/admin/requests` — change status/assignee on any service request
- `/[locale]/admin/users` — approve/suspend users, activate/deactivate, assign roles
- `/[locale]/admin/news` — trilingual news CMS with Draft → Pending Review → Published → Archived
  workflow

## System
- `/api/auth/[...nextauth]` — authentication
- `/api/assistant`, `/api/search`, `/api/captcha`, `/api/reports/requests` — AI assistant chat,
  global search, CAPTCHA challenges, and admin CSV export
- `/manifest.json`, `/sw.js`, `/offline.html` — PWA
- `/sitemap.xml` (generated by `src/app/sitemap.ts`), `/robots.txt`

## Cross-cutting functions implemented
- Trilingual content (AR/EN/VI) with RTL for Arabic
- Server-side RBAC (`src/lib/rbac.ts`)
- Zod validation on every Server Action
- Self-hosted CAPTCHA (`src/lib/captcha.ts`) on registration and emergency forms
- In-memory rate limiting (`src/lib/rateLimit.ts`) on login, registration, and emergency submission
- Audit logging on login, registration, request creation, emergency submission, document
  upload/delete, password change, account deactivation, admin status changes
- Digital membership card with QR code generation (`qrcode` package)
- Mock/pluggable AI assistant
- File upload validation (type + 10MB size limit) via `src/lib/storage.ts`
- Loading/error/not-found states per locale segment
